  • 各个页面的功能

  1. js:页面逻辑实现，（就是如果点击一个图标会跳转到另一个程序或者什么的）
  2. json：负责标题栏和一些状态栏
  3. wxml:管理页面有什么内容
  4. wxss：页面排布，（就是把有的东西排列成自己写想要的样式）

  • 一些.wxml知识（内容）

    1. wxml的注释方式：<！– 等等等 –>
    2. 在wxml中内容单元封装在<view>等等等</view>中
    3. 图片的书写格式<image></image>（图片文件要先新加一个文件夹和page一个层级下，新建images）
    4. 添加图片的方法是在<image src='相对路径（可以去复制，但是要注意反斜杠变化'）</image>，要先在
    5. 文字的书写格式<text>内容</text>
    6. 如果文字的内容中间是<text></text>表示内容是可动态变化的，是个变量。但是这个变量需要在.js文件中的data：{}中来写入
    7. 按钮的书写格式<button>按钮上想写的字</button>
    8. 以上的都是在wxml中添加完成，想要布局更好看就要去wxss中布局
    9. 在标签中两个花括号之间包的叫变量，是可以改变的

  • 一些.wxss知识（样式设计）

  1. wxss注释方法：/* 等等等 */

  2. 如果要布局要先在wxml中的view开始括号里加一个 class =‘等等等’（等等等是自取的，class不能变）

  3. 然后在wxss中中进行样式设计：要先加个点·然后接class里面的名字（ag：.container{}）（在view中就是全局布局）

  4. 如果说是图片的的样式有两种方法

    1. 直接在wxss中写成image{}，大括号里填属性

    2. 在wxml中的<image></image>中也这样在可以加样式class：这样在wxss中也要写样式.等等等

  5. 文本样式的话跟图片一样，也是可以直接在wxss中text{}中加东西

  6. 想要实现点击按钮然后实现某种功能则在.wxss中的<button “属性内容”>中加上限定，

  7. 然后在.js中写上函数，这个函数是要在data同排下的

  • 一些.js知识(函数，变量)

  1. .js中函数的定义方法：

    函数名+“：”+function(“参数列表（可不写）”){“函数内容”}，

    ！注意:两个{}{}之间要有逗号,

  2. .js的data{}就是存放变量的地方，所以在这个地方写上变量，而变量是在.wxml中两个标签的，这个就表示变量。

  3. 图片的路径也可以写成变量，就是src，同样也写在data中

  4. 如果想更新变量内容，比如说获取昵称和头像就在原先定义的函数里去调用，修改变量的方法：this.setData({ })
